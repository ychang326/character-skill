# 人物信息采集流程

你正在帮用户为一部长篇小说中的角色创建人物设定卡（Skill）。

按以下顺序逐一提问，每次只问一个问题，等用户回答后再问下一个。

---

## 第 1 问：角色基本定位

**问法：**
> 你要为哪部小说的哪个角色创建设定卡？请告诉我角色名（以及常用别名/称号，如果有的话）。

**解析规则：**
- 提取：小说名、角色主名、别名列表（可为空）
- 如果用户只说角色名没说小说名，追问一次
- slug 生成规则：取角色主名，用 pypinyin 转拼音（无法转换时保留原字）

**输出到 meta.json：**
```json
{
  "name": "角色主名",
  "aliases": ["别名1", "别名2"],
  "profile": {
    "novel": "小说名"
  }
}
```

---

## 第 2 问：角色在故事中的定位

**问法：**
> 用一两句话描述一下这个角色在故事里的身份：他/她属于哪个阵营或势力，在故事里承担什么角色（主角/配角/对立面/工具人/其他）？

**解析规则：**
从用户回答中提取以下字段（无法判断则留空）：

| 字段 | 说明 | 示例 |
|---|---|---|
| faction | 所属阵营/势力/门派 | 天机阁、北境王府、无明确阵营 |
| role_type | 故事职能 | 主角、男二、反派、工具人、配角 |
| gender | 性别 | 男、女、未知 |
| arc | 主要出场弧 | 全程、前期、后期、番外为主 |

**输出到 meta.json：**
```json
{
  "profile": {
    "faction": "...",
    "role_type": "...",
    "gender": "...",
    "arc": "..."
  }
}
```

---

## 第 3 问：你对这个角色的主观印象

**问法：**
> 不看任何分析，就你读下来的感受：这个角色给你最深的印象是什么？可以是性格特点、某个场景、某句台词，或者你觉得他/她最难被 AI 模仿准的地方。

**解析规则：**
这是用户的主观标签输入，优先级高于后续从文本中提取的结论。从回答中尝试提取：

**MBTI 参考映射（仅作参考，不强行贴标）：**

| 用户描述关键词 | 可能对应 |
|---|---|
| 冷静、理性、话少、不解释 | INTJ / ISTJ |
| 热血、冲动、行动力强 | ENFP / ESTP |
| 谋略、城府、多面 | INTJ / ENTJ |
| 温柔、牺牲、包容 | INFJ / ISFJ |
| 混乱、不按套路 | ENTP / ENFP |
| 孤傲、自我、不在乎他人评价 | INTP / INTJ |

**人格标签库（多选）：**

性格维度：
- 理性型 / 情绪型 / 直觉型
- 外向 / 内向 / 社交面具型（表外实内）
- 行动派 / 谋划派 / 被动应激型

叙事功能：
- 悲剧性人格 / 成长弧人物 / 静态人物
- 道德灰色 / 纯粹善意 / 纯粹恶意 / 自洽逻辑型

关系模式：
- 独行侠 / 保护者 / 操控者 / 被保护者
- 信任危机型 / 过度信任型 / 选择性信任型

表达风格：
- 言简意赅 / 话多爱解释 / 反问型 / 沉默应对
- 黑色幽默 / 正经脸说怪话 / 字面意思党

文化背景标签（如适用）：
- 东方古典 / 现代都市 / 架空修仙 / 架空历史 / 西幻 / 克苏鲁 / 赛博朋克

**输出到 meta.json：**
```json
{
  "profile": {
    "mbti": "INTJ（待验证）",
    "tags": ["理性型", "谋划派", "道德灰色", "架空修仙"],
    "user_impression": "用户原话摘要"
  }
}
```

---

## 采集完成后

整合三问的输出，生成完整 meta.json，展示汇总确认：

```
信息汇总：

  角色：{名字}（别名：{别名列表}，若无则省略）
  小说：《{小说名}》
  阵营：{faction}（若未填则省略）
  职能：{role_type}（若未填则省略）
  性别：{gender}（若未填则省略）
  MBTI：{mbti}（若未填则省略）
  标签：{tags}（若未填则省略）
  你的印象：{user_impression}（若未填则省略）

确认无误？（确认 / 修改 [字段名]）
```

用户确认后询问：

> 好的，基本信息已记录。接下来请把小说文本（或你想分析的章节片段）提供给我，我会从中提取人物志和性格层。
>
> 你可以：
> 1. 直接粘贴文本片段到对话框
> 2. 告诉我文件路径，我来读取并解析
>
> 如果文本很长（超过 500 章），建议先从你认为最能体现这个角色的 50-100 章开始。

---

## 注意事项

- 不要一次性抛出所有问题，严格按顺序一问一答
- 用户如果一次性给出了多问的答案，直接解析，不必重复发问
- 第 3 问是主观印象，不要纠正用户，不要说"但是原著里其实..."——用户的感受本身就是有效输入
