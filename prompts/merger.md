# 增量 Merge Prompt

## 任务

你将收到：
1. 现有的 `profile.md` 内容（人物志）
2. 现有的 `persona.md` 内容（人物性格）
3. 新的原著文本（新章节、番外片段等）

你的任务是判断新内容应该更新哪个部分，并输出增量更新内容。

**原则：只追加增量，不覆盖已有结论。如有冲突，输出冲突提示让用户决定。**

---

## Step 1：分类判断

将新章节/番外内容中的每条信息归类：

| 信息类型 | 归入 |
|---------|------|
| 能力设定、战斗表现、新技能展示 | → profile.md |
| 关系变化、新出场人物的互动 | → profile.md |
| 关键剧情节点、重要选择场景 | → profile.md |
| 番外创作参考（新空白地带/新约束） | → profile.md |
| 说话方式、口头禅、新的表达习惯 | → persona.md |
| 决策行为、情绪反应、人际行为 | → persona.md |
| 两者都有 | → 分别归入 |

---

## Step 2：检查冲突

对比新内容与现有内容：

- 如果新内容**补充**了现有信息（增加了新细节）→ 直接追加
- 如果新内容**确认**了现有信息 → 忽略（不重复写）
- 如果新内容**与现有信息矛盾** → 输出冲突提示：

```
⚠️ 发现冲突：
- 现有：{现有描述}
- 新发现：{新内容描述}
- 来源：{章节位置}

建议：[保留现有 / 更新为新内容 / 两者都保留并标注弧段]
请用户决定。
```

---

## Step 3：生成更新 Patch

对 `profile.md` 的更新，输出格式：
```
=== profile.md 更新 ===

[追加到"能力设定/核心能力"节]
- {新能力描述，附章节出处}

[追加到"关键剧情节点"节]
节点 N（位置：{章节范围}）：{事件} → {选择} → {结果}

[追加到"关系图谱/{关系人}"节]
- {关系变化描述}

[追加到"番外创作参考/可延伸空白地带"节]
- {新发现的创作空间}

[无更新] 或 [以上章节有更新]
```

对 `persona.md` 的更新，输出格式：
```
=== persona.md 更新 ===

[追加到"Layer 2/口头禅与高频词"节]
- 新口头禅："{xxx}"

[追加到"Layer 4/{对应关系维度}"节]
- {新行为描述，附原著场景}

[无更新] 或 [以上章节有更新]
```

---

## Step 4：生成更新摘要

向用户展示：
```
本次更新摘要：
- 来源：{章节范围/番外名称}
- profile.md：追加了 {N} 条新信息（{简要描述}）
- persona.md：追加了 {N} 条新信息（{简要描述}）
- 发现 {N} 处冲突，需要你确认（见上方）

版本将从 {vN} 升级到 {vN+1}。
确认应用更新？
```
