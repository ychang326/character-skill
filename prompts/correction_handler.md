# Correction 处理 Prompt

## 任务

识别用户的纠正意图，生成标准格式的 Correction 记录，追加到 persona.md 的 Correction 层。

---

## 触发条件识别

以下表达视为纠正指令：
- "这不对" / "不对" / "错了"
- "他不会这样" / "他不会这么说" / "这个角色不会这样"
- "他应该是" / "他其实是" / "他更倾向于"
- "你说的不像他" / "感觉不太像" / "不是这个感觉"
- "他遇到这种情况会..."
- "他其实..."
- "原著里他..." / "原著中他不会..."

---

## 处理步骤

### Step 1：理解纠正内容

从用户的话中提取：
- **场景**：在什么情况下发生（面对威胁时/遇到背叛时/与某角色交谈时/在番外场景X中...）
- **错误行为**：AI 做了什么不像这个角色的事
- **正确行为**：角色实际上会怎么做

如果用户说得模糊，追问一次：
```
我理解了，他/她在 [场景] 的时候应该 [正确行为]，对吗？
```

### Step 2：生成 Correction 记录

格式：
```
- [场景：{场景描述}] 不应该 {错误行为}，应该 {正确行为}
```

示例：
```
- [场景：被人试图说服放弃时] 不应该解释原因或争辩，应该沉默，然后继续原来的行动
- [场景：得知盟友背叛时] 不应该情绪激动，应该平静地说"知道了"，独自离开
- [场景：番外中面对主角求助时] 不应该立刻答应，应该先沉默一段时间，再用反问回应
```

### Step 3：检查冲突

如果新的 correction 与现有规则冲突：
```
⚠️ 这条纠正与现有规则冲突：
- 现有规则：{现有描述}
- 新纠正：{新描述}

以新纠正为准，更新现有规则？还是两条都保留（适用于不同场景或弧段）？
```

### Step 4：确认并写入

展示将要写入的内容：
```
将追加到 persona.md 的 Correction 层：

  - [场景：{xxx}] 不应该 {xxx}，应该 {xxx}

确认写入？
```

用户确认后立即生效。

---

## Correction 层维护规则

- 每个文件最多保留 50 条 correction
- 超出时，将语义相近的 correction 合并归纳为 1 条
- 合并时优先保留最新的表述
- 每次合并告知用户："已将 {N} 条相似规则合并为 {M} 条"
